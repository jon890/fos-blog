## ADR-031. ADR 파일 분리 — 단일 adr.md → docs/adr/NNN-slug.md (plan052)

- **결정**: 단일 `docs/adr.md`(29개 ADR, 592줄)를 `docs/adr/NNN-slug.md` 1파일 = 1ADR 구조로 분해한다.
  `docs/adr/README.md` 가 분류별 INDEX 라우터 역할을 한다.
  번호를 파일명에 유지(`NNN-slug.md`)해 `ADR-NNN` 텍스트 참조가 파일명으로 찾아지도록 보존한다.
- **맥락**: 단일 `adr.md` 에 새 ADR 을 추가할 때마다 파일 끝줄(신규 ADR + 인덱스 카운트)에서 머지 충돌이 반복 발생했다.
  plan051(PR #176)과 본 plan052 가 실측으로 충돌 위험을 겪었다.
  파일-per-ADR 구조는 각 ADR 이 독립 파일이라 병렬 PR 이 서로 다른 파일에 쓰므로 충돌 표면이 구조적으로 제거된다.
  INDEX(`README.md`) 는 번호순으로 인접 추가되므로 잔여 충돌 면적이 최소화된다.
- **대안 기각**:
  - **단일 adr.md 유지** 기각 — PR 마다 끝줄 충돌이 지속된다. 기여자가 늘거나 plan 이 병렬화될수록 비용이 증가한다.
  - **신규 ADR 만 분리 + 기존 단일 유지** 기각 — 단일+분리 혼재로 발견성이 저하된다. 참조 패턴도 두 가지가 공존해 유지보수 혼란이 생긴다.
- **트레이드오프**:
  - 일회성 비용: 29개 파일 분리 + `docs/`·`.claude/`·`CLAUDE.md` 범위 참조 갱신.
  - 이후 충돌 비용: INDEX 에 한 줄 추가만 남아, 번호순 인접 삽입이라 자동 해소.
  - 결번(012 등)은 파일을 만들지 않고 INDEX 에 보존 정책만 명시한다.
